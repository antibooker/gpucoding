# CUDA VectorAdd Demo
基于 CUDA C 实现 GPU 并行向量加法，对比 CPU 与 GPU 在不同数据规模下的计算效率，熟悉 CUDA 核函数、内存管理与错误检查。

## 运行结果示例
```
===== 数据规模：1024 个元素 =====
CPU 计算耗时：0.00 ms
GPU 计算耗时：0.97 ms
计算结果验证：前2个元素结果为 0, 3

===== 数据规模：10240 个元素 =====
CPU 计算耗时：0.03 ms
GPU 计算耗时：2.67 ms
计算结果验证：前2个元素结果为 0, 3

===== 数据规模：102400 个元素 =====
CPU 计算耗时：0.26 ms
GPU 计算耗时：12.98 ms
计算结果验证：前2个元素结果为 0, 3
```
> *注：所有GPU存在启动与调度开销（只是在线共享 GPU（资源共享、有虚拟化层）的调度开销，通常比本地独立 GPU 更大），本地环境下大数据量GPU并行优势会显著体现，在本次的代码实践中我还发现，在数据量较小时相比于调用1个块，调用4096个块的计算时间更长，这是反直觉的，通过学习我了解到gpu的启动和调度同样需要时间，对于较小的数据量，其启动和调度的时间可能比实际运算的时间更长。*

## 技术要点
- CUDA 核函数编写与调用
- 主机/设备内存分配与数据拷贝
- 并行线程块/网格配置
- 计时与性能对比
- 错误处理机制

## 如何运行
```bash
nvcc vector_add.cu -o vecadd
./vecadd
# 也可通过CMake一键构建：cmake . && make
```
